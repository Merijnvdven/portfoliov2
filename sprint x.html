Sprint X

Wat heb ik gedaan?

Voor mijn sprint x had ik het idee om een site te maken met 4 simpele minigames erop. Hiermee kan ik mezelf nog verder ontwikkelen met development. De minigames liet ik vervolgens usertesten door een paar gebruikers. Zodat ik later aanpassingen kon maken om de spellen zo gebruiksvriendelijker te maken. Mijn sprint x is dus vooral gericht naar interactieve media en development.





Wat heb ik geleerd?

Zoals hierboven al aangegeven was de essentie van deze sprint mijzelf nog verder ontwikkelen in development en ux (de usertests). In de meeste spellen zit vooral tijd in javascript in minder in html en css (op memory na). Bij het memory spel heb ik veel met css gewerkt. Ik heb daar geleerd hoe je de kaarten horizontaal en verticaal in een rij krijgt. Dat was best lastig omdat tussen elke kaart evenveel ruimte moet zitten. De usertests waren handig om zo de site gebruiksvriendelijker en ook qua design mooier te maken. Ik heb door de feedback de site bijgewerkt naar de wensen van de gebruiker. Kort samengevat was dat speluitleg en het uiterlijk van de site. Hieronder zijn een versie 1 en 2 te zien van de site





Wat kon beter?

Development: Het uiterlijk van zowel de site en de spellen had ik meer tijd in kunnen steken zodat het visueel er beter uit komt te zien. interactieve media: Bij de usertest had ik maar een beperkt aantal vragen. Dat had ik iets kunnen opschalen om zo een beter beeld te krijgen wat de gebruiker mist aan de site of graag had willen zien.
